Heart Disease MLOps

End-to-end MLOps architecture

From raw UCI Cleveland data to a monitored prediction service running on Kubernetes.

Pipeline stages

Data & Features
UCI Cleveland data cleaned, binarized, and enriched with derived clinical features (RPP, HRR, oldpeak x slope).
Training & Tracking
Six model / feature-set combinations grid-searched and logged to MLflow with ROC and confusion-matrix plots.
Packaging
Best run (logistic regression + clinical features, test ROC-AUC 0.8817) exported to a versioned joblib pipeline.
CI/CD
GitHub Actions lints, runs all 39 tests, smoke-trains, and builds the Docker image on every push.
Kubernetes
Two-replica FastAPI deployment behind a LoadBalancer, scraped by Prometheus and visualized in Grafana.
Client
A clinician (or this dashboard) sends patient features to POST /predict and gets back a prediction and probability.

Source: github.com/axle-bits/MLOP_Assignment_1
